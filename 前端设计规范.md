# CogniFlow 前端设计规范

## 1. 设计参考

以《明日方舟》官网（ak.hypergryph.com）的设计语言为参考，核心特征：

- **深色为主**，高明度点缀色形成强对比
- **几何装饰**，三角、六边形、斜线切割等工业感元素
- **克制用色**，大面积暗色 + 小面积高饱和点缀
- **秩序感强**的结构化布局，大量留白

CogniFlow 有两套前端，风格策略差异化：

| | 用户端 Widget | 后台管理系统 |
|------|------|------|
| 基调 | 浅色 / 跟随宿主网站 | 深色，完整 Arknights 风格 |
| 原因 | 嵌入电商官网，不能喧宾夺主 | 独立应用，沉浸式运营工作台 |
| 设计语言 | 轻量版：几何线条装饰 + 克制用色 | 完整版：全深色主题 + 工业科幻感 |

│  辅助强调   #F0A050    琥珀色，警告、高优先级标记              │

## 4. 几何装饰语言（Arknights 风格核心）

### 4.1 斜线切割

卡片和区块的角部使用 45° 斜线切割，而非圆角或直角，形成锐利的工业感。

```
  ┌──────────────────────
  │                      
  │   卡片内容区域        
  │                      
  │                      
  └──────────────────────
  
  右上角斜切：clip-path: polygon(0 0, calc(100% - 12px) 0, 100% 12px, 100% 100%, 0 100%);
```

**使用场景**：后台管理的卡片、Modal 弹窗、侧边栏菜单项。Widget 不用斜切（保持亲和力）。

### 4.2 六边形标识

Logo 和关键状态图标使用六边形底座，致敬 Arknights 的源石（Originium）符号。

```
     ╱  ╲
    ╱    ╲
   ╱      ╲      用于：Agent 状态指示器
   ╲      ╱           加载动画
    ╲    ╱            系统 Logo
     ╲  ╱
      ╲╱
```

### 4.3 细线几何分割

用 1px 的细线组成菱形、三角形网格装饰，用于空白区域的背景纹理。

```css
.geo-pattern {
  background-image:
    linear-gradient(45deg,  transparent 49.5%, rgba(0,212,228,0.06) 49.5%,
                            rgba(0,212,228,0.06) 50.5%, transparent 50.5%),
    linear-gradient(-45deg, transparent 49.5%, rgba(0,212,228,0.06) 49.5%,
                            rgba(0,212,228,0.06) 50.5%, transparent 50.5%);
  background-size: 40px 40px;
}
```

**使用场景**：后台管理的登录页背景、空状态占位图、仪表盘顶部装饰。

## 5. 卡片与容器

### 5.1 后台卡片

```css
.card {
  background: var(--bg-secondary);
  border: 1px solid var(--border-default);
  /* 无圆角，用 clip-path 做斜切角 */
  clip-path: polygon(0 0, calc(100% - 10px) 0, 100% 10px, 100% 100%, 0 100%);
  padding: 20px;
}

/* 悬浮态：边框着色 + 微弱辉光 */
.card:hover {
  border-color: var(--border-accent);
  box-shadow: var(--glow-accent);
}
```

**三种卡片层级**：
- **一级卡片**（仪表盘统计卡片）：加顶部 2px 的 `--accent-primary` 色条
- **二级卡片**（列表中的条目）：默认边框，hover 时边框着色
- **三级卡片**（表格行、信息块）：仅背景色区隔，无边框

### 5.2 Widget 气泡

```css
/* Bot 消息气泡 —— 左侧，灰色底 */
.bubble-bot {
  background: var(--bg-bot-bubble);
  border-radius: 4px 12px 12px 12px;   /* 左上直角，其余圆角 */
  max-width: 80%;
}

/* 用户消息气泡 —— 右侧，青色底 */
.bubble-user {
  background: var(--bg-user-bubble);
  color: #FFFFFF;
  border-radius: 12px 4px 12px 12px;   /* 右上直角，其余圆角 */
  max-width: 80%;
}
```

**直角朝向说话者**：这是 Arknights UI 中斜切逻辑的圆角替代方案——直角指向消息来源，视觉上形成方向感。

### 6.1 主按钮

```css
.btn-primary {
  background: var(--accent-primary);
  color: #0A0E14;                    /* 深色文字，最大化对比 */
  border: none;
  padding: 8px 24px;
  font-size: 14px;
  font-weight: 600;
  clip-path: polygon(0 0, calc(100% - 8px) 0, 100% 8px, 100% 100%, 0 100%);
  transition: all 0.2s ease;
}

.btn-primary:hover {
  background: var(--accent-deep);
  box-shadow: var(--glow-accent);
}

.btn-primary:disabled {
  background: var(--text-disabled);
  color: var(--text-secondary);
  clip-path: none;
}
```

### 6.2 次按钮 / 幽灵按钮

```css
.btn-secondary {
  background: transparent;
  color: var(--accent-primary);
  border: 1px solid var(--border-accent);
  padding: 8px 24px;
  clip-path: polygon(0 0, calc(100% - 8px) 0, 100% 8px, 100% 100%, 0 100%);
}

.btn-secondary:hover {
  background: rgba(0, 212, 228, 0.08);
}
```

### 6.3 危险按钮

```css
.btn-danger {
  background: transparent;
  color: var(--accent-danger);
  border: 1px solid rgba(224, 85, 106, 0.3);
}

.btn-danger:hover {
  background: rgba(224, 85, 106, 0.1);
  border-color: var(--accent-danger);
}
```

### 6.4 Widget 按钮（轻量）

Widget 内不使用斜切按钮（避免过于风格化），用微圆角：

```css
.widget-btn {
  border-radius: 6px;
  padding: 6px 16px;
  font-size: 13px;
  transition: background 0.15s ease;
}
```

## 7. 图标

使用 `lucide-react` 图标库（线性图标，线条粗细 2px，风格克制，与 Arknights 的工业感匹配）：

```jsx
import { Bot, FileText, BarChart3, Settings, AlertTriangle } from 'lucide-react';
```

**图标尺寸体系**：

| 场景 | 尺寸 | 示例 |
|------|------|------|
| 内联图标（按钮内文字旁） | 16px | 按钮中的图标 |
| 标准图标（菜单、列表操作） | 20px | 侧边栏菜单 |
| 状态指示器 | 24px | Agent 状态、系统健康 |
| 装饰图形（空状态、登录页） | 48px+ | 无数据占位 |

**着色规则**：图标默认继承文字色（`currentColor`），仅在选中态和告警态使用强调色。

## 9. 动效

**原则**：快进慢出，不拖沓。动画时长控制在 150-300ms。

| 场景 | 时长 | 缓动 | 说明 |
|------|------|------|------|
| hover 状态切换 | 150ms | ease | 按钮、卡片、菜单项 |
| Modal / Drawer 进出 | 250ms | ease-out | 弹窗从下往上滑出 |
| 侧边栏收缩 | 200ms | ease | 菜单折叠 |
| 骨架屏闪烁 | 1.5s | linear 循环 | 加载占位 |
| 消息气泡出现 | 200ms | ease-out | translateY(8px) → 0 + opacity |
| 流式输出光标 | 1s | steps(1) 闪烁 | `|` 字符间隔闪烁，模拟打字 |
| 辉光脉冲（系统状态） | 2s | ease-in-out 循环 | 正常运行 Agent 的呼吸灯 |

```css
/* 消息气泡入场 */
@keyframes bubble-in {
  from { opacity: 0; transform: translateY(8px); }
  to   { opacity: 1; transform: translateY(0); }
}

.bubble-bot, .bubble-user {
  animation: bubble-in 0.2s ease-out;
}

/* 呼吸灯 —— Agent 在线指示 */
@keyframes breathe {
  0%, 100% { opacity: 1;   box-shadow: 0 0 4px rgba(0,212,228,0.3); }
  50%     { opacity: 0.6;  box-shadow: 0 0 12px rgba(0,212,228,0.6); }
}

.status-dot-online {
  width: 8px; height: 8px;
  background: var(--accent-primary);
  clip-path: polygon(50% 0%, 100% 25%, 100% 75%, 50% 100%, 0% 75%, 0% 25%);
  animation: breathe 2s ease-in-out infinite;
}
```

**Widget 动效限制**：Widget 不使用大幅位移动画（嵌入后可能与宿主网站动画冲突），只做 `opacity` 和微小的 `translateY(≤8px)`。

## 10. 状态指示与反馈

### 10.1 Agent 状态

```
在线      ●  #4ECDC4  呼吸动画   │  告警      △  #F0A050  脉动
离线      ●  #4A5468  无动画     │  异常      ×  #E0556A  闪烁
维护中    ◇  #8B95A5  慢呼吸     │
```

### 10.2 工具执行状态（Widget 内对用户可见）

```
思考中  "..." 三点跳动
查询中  "正在为您查询订单信息…"  + 六边形旋转动画
成功    "已查到您的订单" + ✓ 图标
失败    "暂时无法查询"  + 重试按钮 + 转人工入口
```

### 10.3 流式输出指示

Bot 回复以打字机效果逐字出现，光标 `▌` 在尾部闪烁。生成完成后光标消失，3s 后出现反馈按钮（有帮助/没帮助）。

## 13. 前端技术栈

```
框架：       React 18 + TypeScript
构建工具：   Vite
样式方案：   Tailwind CSS + CSS Variables（自定义主题）
组件库：     自研组件（参考 shadcn/ui 的代码分发模式）
图标：       lucide-react
状态管理：   Zustand（轻量，不引入 Redux 的复杂度）
HTTP 客户端：fetch + 自定义 hook 封装
WebSocket： 原生 WebSocket + 重连逻辑封装
路由（后台）：React Router v6
表格（后台）：TanStack Table
表单验证：   Zod + react-hook-form
动效：       CSS transition/animation + framer-motion（仅复杂场景）
```
